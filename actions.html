Button Variants

Six semantic variants covering every action priority from primary CTAs to destructive operations, plus accent color variants.

Default
Accent Solid
Accent Tonal
Disabled
When to use / avoid
When to use
  • Primary for main CTA, Secondary for supporting actions, Ghost for low-emphasis.
Avoid when
  • Using Primary for more than one action per section, or Danger for non-destructive actions.
<button class="btn btn-primary">Primary</button>
<button class="btn btn-secondary">Secondary</button>
<button class="btn btn-ghost">Ghost</button>
<button class="btn btn-danger">Danger</button>
<button class="btn btn-tonal">Tonal</button>
<button class="btn btn-elevated">Elevated</button>

<!-- Accent color variants -->
<button class="btn btn-accent2">Accent 2</button>
<button class="btn btn-accent3">Accent 3</button>

<!-- Tonal accent variants -->
<button class="btn-tonal-accent2">Tonal Accent 2</button>
<button class="btn-tonal-accent3">Tonal Accent 3</button>

Button Sizes

Three sizes for different layout contexts. Default handles most cases.

When to use / avoid
When to use
  • Small for inline/table contexts, Large for hero CTAs.
Avoid when
  • Mixing sizes in the same button row.
<button class="btn btn-primary btn-sm">Small</button>
<button class="btn btn-primary">Default</button>
<button class="btn btn-primary btn-lg">Large</button>

With Icons & Loading

Icons reinforce meaning. Use 16px or 20px SVG line icons at 1.5 stroke weight.

With Icons
Loading State
Click to simulate loading
When to use / avoid
When to use
  • Icons paired with labels for clarity; icon-only for well-known actions (search, close).
Avoid when
  • Icon meaning is ambiguous without a label; always add aria-label for icon-only buttons.
<!-- Icon + label -->
<button class="btn btn-primary" style="display:inline-flex;align-items:center;gap:8px;">
  <svg width="16" height="16" ...>...</svg>
  Add Item
</button>

<!-- Icon only -->
<button class="btn btn-secondary btn-icon" aria-label="Search">
  <svg width="16" height="16" ...>...</svg>
</button>

<!-- Loading state -->
<button class="btn btn-primary" disabled>
  <span class="spinner"></span> Saving...
</button>

FAB (Floating Action Button)

Floating action buttons surface the primary action of a screen. Position at the bottom-right corner.

Variants

When to use / avoid
When to use
  • Single primary action per screen.
Avoid when
  • Page has multiple competing primary actions.
<!-- Standard FAB -->
<button class="fab" aria-label="Create">
  <svg ...></svg>
</button>

<!-- Small / Extended / Accent variants -->
<button class="fab fab-sm">...</button>
<button class="fab fab-extended">...Label</button>
<button class="fab fab-accent2">...</button>
<button class="fab fab-accent3">...</button>

<!-- Speed Dial -->
<div class="speed-dial">
  <div class="speed-dial-actions">
    <div class="speed-dial-item" style="--i:1">
      <span class="speed-dial-label">Edit</span>
      <button class="fab fab-sm" aria-label="Edit">
        <svg ...></svg>
      </button>
    </div>
    <div class="speed-dial-item" style="--i:2">
      <span class="speed-dial-label">Share</span>
      <button class="fab fab-sm" aria-label="Share">...</button>
    </div>
  </div>
  <button class="fab speed-dial-trigger"
          onclick="toggleSpeedDial(this)">
    <svg class="speed-dial-icon-open">...</svg>
    <svg class="speed-dial-icon-close">...</svg>
  </button>
</div>

Segmented Buttons

Mutually exclusive selection within a compact group. Use for view modes or period filters.

Text Segmented
Icon Segmented
When to use / avoid
When to use
  • View mode switches, time period filters.
Avoid when
  • More than 5 options, or options aren't mutually exclusive.
<div class="seg-group" role="group" aria-label="Time period">
  <button class="seg-btn active" onclick="setSeg(this)">Day</button>
  <button class="seg-btn" onclick="setSeg(this)">Week</button>
  <button class="seg-btn" onclick="setSeg(this)">Month</button>
  <button class="seg-btn" onclick="setSeg(this)">Year</button>
</div>

Social Buttons

OAuth and social login buttons. Outlined style with provider icon and label.

When to use / avoid
When to use:
  • Auth pages offering third-party login (Google, GitHub, etc.).
Avoid when:
  • Non-auth contexts — use standard buttons.
<div class="social-btns">
  <button class="social-btn">
    <svg>...</svg> Continue with Google
  </button>
  <button class="social-btn">
    <svg>...</svg> Continue with GitHub
  </button>
</div>

Or-Divider

Horizontal rule with centered text label. Used to separate auth methods.

or
When to use / avoid
When to use:
  • Between primary form and social login options.
Avoid when:
  • Standard content separation — use a plain divider.
<div class="or-divider">or</div>
