Alerts

Inline feedback messages for informational, success, warning, and error states.

New features available
Several updates have been released. Review the changelog for details on what changed.
Changes saved successfully
Your profile has been updated and the changes are now live.
Storage nearing limit
You have used 87% of your available storage. Consider archiving or deleting old files.
Failed to send message
There was a network error. Check your connection and try again.
When to use / avoid
When to use
  • Persistent inline feedback for important status changes
Avoid when
  • Feedback is transient — use snackbar instead
<div class="alert alert-info" role="alert">
  <span class="alert-icon">...</span>
  <div class="alert-content">
    <div class="alert-title">New features available</div>
    <div class="alert-desc">Several updates have been released.</div>
  </div>
</div>
<div class="alert alert-success" role="alert">...</div>
<div class="alert alert-warning" role="alert">...</div>
<div class="alert alert-error" role="alert">...</div>

Snackbar / Toast

Transient messages anchored to the bottom of the viewport. Auto-dismiss after 3 seconds.

When to use / avoid
When to use
  • Brief confirmations of completed actions
Avoid when
  • Information requires user action to resolve — use alert
<div class="snackbar" id="snackbar" role="status" aria-live="polite">
  <span id="snackbarMsg"></span>
  <button class="snackbar-action" id="snackbarAction"></button>
</div>

<!-- Trigger -->
<button onclick="showSnackbar('Changes saved.', 'Undo')">Save</button>

Tooltips

Hover-triggered contextual labels. Use sparingly — not for essential information.

When to use / avoid
When to use
  • Supplementary labels on icon-only buttons
Avoid when
  • Information is essential to the task — show it inline
<span class="tooltip">
  <button class="btn btn-ghost btn-icon"
    data-tooltip="Open keyboard shortcuts"
    aria-label="Keyboard shortcuts">
    <!-- icon SVG -->
  </button>
</span>

Dot Loader

Bouncing-dot indicator for typing states, inline loading, and conversational UIs.

Sizes
dot-loader-sm
default
dot-loader-lg
Inline with text

When to use / avoid
When to use
  • Typing or "thinking" indicators in chat and conversational UIs.
  • Inline loading where a full spinner feels too heavy.
Avoid when
  • Loading an entire page or large section — use a spinner instead.
  • Progress is deterministic — use a progress bar.
