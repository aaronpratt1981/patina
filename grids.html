When to use / avoid
When to use
  • Even distribution of cards, forms, or content blocks
Avoid when
  • Content items vary greatly in size or count — use auto-fill grids instead

When to use / avoid
When to use
  • Product grids, card galleries, or any collection with variable item count
Avoid when
  • You need exact column control — use fixed grids

Asymmetric Grids

Common layout patterns using custom grid-template-columns.

<!-- Content + Sidebar -->
<div style="display: grid; grid-template-columns: 1fr 300px; gap: 16px;">
  <div>Main Content</div>
  <div>Sidebar 300px</div>
</div>

<!-- Sidebar + Content -->
<div style="display: grid; grid-template-columns: 240px 1fr; gap: 16px;">
  <div>Sidebar 240px</div>
  <div>Main Content</div>
</div>

<!-- Three-column -->
<div style="display: grid; grid-template-columns: 200px 1fr 280px; gap: 16px;">
  <div>Nav 200px</div>
  <div>Content</div>
  <div>Aside 280px</div>
</div>
When to use / avoid
When to use
  • Page layouts with a main content area and a sidebar or aside
Avoid when
  • All columns should be equal width — use fixed-column grids

Nested Grids

Grids can be nested for complex layouts. Inner grids inherit gap from their context.

B
C
D
Full Height
<div class="grid-2">
  <!-- Left column: inner 2-column grid -->
  <div>
    <div class="grid-2">
      <div>A</div>
      <div>B</div>
      <div>C</div>
      <div>D</div>
    </div>
  </div>
  <!-- Right column: single tall cell -->
  <div>Full Height</div>
</div>
When to use / avoid
When to use
  • Complex layouts where an inner region needs its own column structure
Avoid when
  • A single grid with spanning cells can achieve the same result

When to use / avoid
When to use
  • Adjusting density — tight gaps for compact data, loose gaps for visual breathing room
Avoid when
  • The default 16px gap already works — avoid overriding without reason
