Sidebar
The sidebar is rendered by sidebar.js and visible on this page. See sidebar.js for the data-driven approach.

The sidebar on this page is injected by sidebar.js at runtime. It reads a central navItems array and auto-highlights the current page. Navigation icons are inline SVG, and optional badge counts are supported per item.

Sidebar Toggle
A 56x56 hamburger button positioned in the top-right corner of the sidebar header. Click to collapse or expand the sidebar. When collapsed, it appears fixed at the top-left of the viewport.
Waffle Button
A 56x56 app-switcher button positioned in the top-left corner of the sidebar header. Opens a grid menu of available applications.

Sidebar Profile
User avatar, name, and email anchored at the bottom of the sidebar. Highlights on hover.
When to use / avoid
When to use
  • Primary navigation for multi-page applications with persistent sections.
Avoid when
  • Single-page apps with few routes — use tabs or top nav.

When to use / avoid
When to use
  • Page-level header identifying the current section with its sidebar icon.
  • Component and reference pages that benefit from a persistent, sticky title bar.
Avoid when
  • Content already has a clear heading — avoid redundant titling.
  • Pages where a breadcrumb or inline heading is more appropriate.

Tabs
Horizontal tab navigation with an active underline indicator. Use for switching between views within the same context.

Tab panel content renders here. The active indicator uses a 2px rust-colored underline.

When to use / avoid
When to use
  • Switching views within same context.
Avoid when
  • Navigation between separate pages.
<div class="tabs" role="tablist">
  <button class="tab active" role="tab" aria-selected="true" onclick="setTab(this)">General</button>
  <button class="tab" role="tab" aria-selected="false" onclick="setTab(this)">Security</button>
  <button class="tab" role="tab" aria-selected="false" onclick="setTab(this)">Billing</button>
</div>

Search Bar
Inline search input with leading icon and optional clear button. Gains a focus ring on interaction.
When to use / avoid
When to use
  • Filtering content or navigating to pages.
Avoid when
  • Filtering a short list — use chips.
<div class="search-bar" style="max-width:400px;">
