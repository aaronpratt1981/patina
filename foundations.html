When to use / avoid
When to use
  • Reference tokens for all color values. Use semantic tokens (green, red, amber) for status.
Avoid when
  • Hard-coding hex values directly in components.
<div style="color: var(--accent)">Primary action</div>
<div style="background: var(--accent-bg); border: 1px solid var(--accent-border)">
  Accent tinted surface
</div>
<div style="color: var(--success)">Success status</div>
<div style="color: var(--danger)">Error status</div>
<div style="color: var(--warning)">Warning status</div>

Inline Code

Use var(--accent) for the primary brand color and .btn-primary for CTAs.

Code Block
.card {
  background: var(--white);
  border: 1.5px solid var(--border);
  border-radius: var(--radius-lg);
  padding: 24px;
}
When to use / avoid
When to use
  • Display for headings, Inter for body/UI, Mono for code and data.
Avoid when
  • Mixing display typeface into body text or using body font for headings.
<h1 class="t-display t-display-lg">Display Large</h1>
<p class="t-body">Body text in Inter.</p>
<span class="t-small">Small supporting text</span>
<span class="t-label">Label text</span>
<code class="t-mono">Monospaced code</code>

When to use / avoid
When to use
  • Use spacing tokens for all margin, padding, and gap values to maintain consistent rhythm.
Avoid when
  • Arbitrary pixel values that don't align with the 4px grid.
<!-- Spacing tokens: 4, 8, 12, 16, 20, 24, 28, 32, 40, 48, 56, 64 -->

Elevation / Shadows

Five levels of elevation expressed through layered box-shadows with warm undertones.

--shadow-xs
--shadow-sm
--shadow-md
--shadow-lg
--shadow-focus
When to use / avoid
When to use
  • Higher elevation for popovers and modals; lower for cards and subtle lifts.
Avoid when
  • Flat surfaces that don't need visual depth — use borders instead.
<div style="box-shadow: var(--shadow-xs);">Extra small</div>
<div style="box-shadow: var(--shadow-sm);">Small</div>
<div style="box-shadow: var(--shadow-md);">Medium</div>
<div style="box-shadow: var(--shadow-lg);">Large</div>
<div style="box-shadow: var(--shadow-focus);">Focus ring</div>

When to use / avoid
When to use
  • Match radius to component size: small for inputs, large for cards, full for pills and avatars.
Avoid when
  • Using arbitrary radius values outside the scale.
<div style="border-radius: var(--radius-xs);">6px</div>
<div style="border-radius: var(--radius-sm);">10px</div>
