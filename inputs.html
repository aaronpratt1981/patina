Text Fields

Standard single-line text input in six states covering all interaction scenarios.

Default
With Value
With Hint
Letters, numbers, and underscores only.
Error State
Enter a valid email address.
Password
Disabled
When to use / avoid
When to use
  • Short-form data entry (names, emails, URLs).
Avoid when
  • Long-form content — use textarea instead.
<div class="field">
  <label class="field-label">Full Name</label>
  <input type="text" class="field-input" placeholder="Jane Smith">
  <div class="field-hint">Helper text goes here.</div>
</div>

<!-- Error state -->
<div class="field field-error">
  <label class="field-label">Email</label>
  <input type="email" class="field-input" value="not-an-email">
  <div class="field-hint">Enter a valid email address.</div>
</div>

Select

Native select element with optgroup support for categorized options.

Default
With Selection
When to use / avoid
When to use
  • 3–10 predefined options.
Avoid when
  • Options exceed 10 — use picklist with search.
<div class="field">
  <label class="field-label">Role</label>
  <select class="field-input">
    <option>Viewer</option>
    <option selected>Editor</option>
    <option>Admin</option>
  </select>
</div>

Textarea

Multi-line text input for longer free-form content.

Markdown is supported.
When to use / avoid
When to use
  • Multi-line freeform text: descriptions, notes, comments.
Avoid when
  • Short single-line values — use a text field.
<div class="field">
  <label class="field-label">Description</label>
  <textarea class="field-input" rows="4" placeholder="Write a brief description..."></textarea>
  <div class="field-hint">Markdown is supported.</div>
</div>

Text Toolbar

Formatting toolbar for rich text editing. Attach above a textarea to provide inline styling controls.

Full Toolbar
Minimal Toolbar
When to use / avoid
When to use
  • Rich text fields: descriptions, comments, messages with formatting.
Avoid when
  • Plain text is sufficient — use a standard textarea.
<div class="field">
  <label class="field-label">Description</label>
  <div class="text-toolbar">
    <select class="text-toolbar-select">
      <option>Paragraph</option>
      <option>Heading 1</option>
    </select>
    <div class="text-toolbar-sep"></div>
    <button class="text-toolbar-btn" title="Bold"><strong>B</strong></button>
    <button class="text-toolbar-btn" title="Italic"><em>I</em></button>
    <button class="text-toolbar-btn" title="Underline">U</button>
    <div class="text-toolbar-sep"></div>
    <button class="text-toolbar-btn" title="Bulleted list">...</button>
    <button class="text-toolbar-btn" title="Link">...</button>
    <button class="text-toolbar-btn" title="Code">...</button>
  </div>
  <textarea class="field-input" rows="5"
    style="border-radius:0 0 var(--radius-sm) var(--radius-sm);"></textarea>
</div>

Radio Buttons

Single selection from a set of mutually exclusive options.

Pricing Tier
Essentials
Free — Up to 3 projects
Studio
$12 / mo — Unlimited projects, team collaboration
Enterprise
Custom — SSO, audit logs, SLA
When to use / avoid
When to use
  • 2-5 mutually exclusive options that benefit from visible context.
Avoid when
  • Many options — use a select or picklist instead.
<div class="radio-option selected" onclick="selectPlan(this)">
  <input type="radio" name="plan" value="essentials" checked style="accent-color:var(--accent);">
  <div class="radio-option-info">
    <div class="radio-option-name">Essentials</div>
    <div class="radio-option-price">Free — Up to 3 projects</div>
  </div>
</div>

Checkboxes

Multi-selection with independent checked, unchecked, and disabled states.

When to use / avoid
When to use
  • Multiple independent boolean choices (preferences, permissions).
Avoid when
  • Only one option exists — use a switch instead.
<label style="display:flex;align-items:center;gap:10px;cursor:pointer;font-size:0.9rem;">
  <input type="checkbox" checked style="accent-color:var(--accent);width:16px;height:16px;">
  Receive product updates
</label>

<label style="cursor:not-allowed;color:var(--text-dim);">
  <input type="checkbox" disabled style="width:16px;height:16px;">
  Required terms (locked)
</label>

Switches / Toggles

Binary on/off controls for settings and preferences.

Dark Mode
Apply dark theme across the interface
Email Digest
Receive a weekly summary of activity
Beta Features
Available on Studio plan and above
When to use / avoid
When to use
  • Instant on/off toggles for settings and preferences.
Avoid when
  • Change requires a form submission — use checkbox instead.
<div class="switch-row">
  <div class="switch-info">
    <div class="switch-label-text">Dark Mode</div>
    <div class="switch-sub">Apply dark theme across the interface</div>
  </div>
  <div class="toggle-track" onclick="this.classList.toggle('active')">
    <div class="toggle-thumb"></div>
  </div>
</div>

<!-- Active state -->
<div class="toggle-track active" onclick="this.classList.toggle('active')">
  <div class="toggle-thumb"></div>
</div>

Slider

Range input for continuous numerical values with live value display.

60%
35%
When to use / avoid
When to use
  • Continuous numeric ranges with immediate visual feedback.
Avoid when
  • Exact numeric input is needed — use a text field.
<div class="field">
  <label class="field-label">Volume</label>
  <div class="slider-row">
    <input type="range" class="slider" min="0" max="100" value="60"
      oninput="updateSlider(this,'val','%')">
    <span class="slider-val" id="val">60%</span>
  </div>
</div>
